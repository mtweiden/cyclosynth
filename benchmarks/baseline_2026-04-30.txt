  improve this by filling cores even when |L| is small.

=== Strategies attempted in this session ===

Strategy 3 (BKZ-3): Implemented bkz3_reduce to minimize b[0] (shortest lattice vector).
  REGRESSION: 4x more phase2_pq calls, 4-6x slower total.
  Root cause: schnorr_euchner solves z0 analytically by dividing by r_mat[0][0]=‖b[0]‖.
  BKZ-3 makes b[0] shorter → floating-point amplification → solutions missed → synthesis
  tries higher t values → call count explodes.  REVERTED.

Strategy 5 (conditioned a2/c2 centers): Implemented per-(a1,c1) conditioned centers.
  No measurable benefit.  Analysis: conditioned center ≈ global center for the central
  (a1,c1) pairs where solutions live.  For off-center pairs, better centering doesn't
  help because those pairs don't contain solutions.

Tight CenteredRange bounds (sqrt(rem) instead of max_outer): Causes correctness bug.
  CenteredRange(center, radius) only covers [center-radius/2, center+radius/2], not
  [-radius, radius].  With non-zero conditioned center, some valid candidates are missed.

Strategy 6 (t-parity gap): Analysis shows t_inner is always 24 for eps=0.001 (all t ≥ 25
  give t_prime = t-24 via ceil(t-24.912)).  k_inner = 13 for all t ≥ 25.  Consecutive t
  values use DIFFERENT prefix sets (L_3 vs L_4) not the same k_inner.  Strategy 6 premise
  is incorrect; does not apply to this implementation.

=== Net changes from Run 3 to Run 4 ===

- Removed non-guarded debug eprintln! calls from synthesize(), try_at_lde(), dc_search()
- Added sequential fallback: Phase 1b runs sequentially when pairs.len() < n_threads*4
- No measurable performance change (within noise of Run 3)

=== Notes (Run 5) ===

- Strategy A changes: schnorr_euchner signature extended with y_inner, dot_outer,
  threshold_sq; return type changed from Vec<[i64;4]> to Option<[i64;4]>. z2 and z1
  loops now iterate outward from the CVP center. Alignment Cauchy-Schwarz prune fires
  per z2 (skips z1 loop) and per z1 (skips z0 computation) when the achievable
  |dot| range [partial_eff ± max_align] lies entirely within (-threshold, threshold).
  z1 prune uses partial_eff = total_center + z1_dot + z0_center_dot (incorporates
  z0's analytic center contribution for a tighter bound). phase2_pq signature changed
  to take y_inner, dot_outer, threshold_sq instead of the full y vector.
  phase1_enumerate inner closure updated to pass these values.

- Speedup vs Run 4 (t=1): Rz_0.3_tight 940ms → 386ms (2.4x), Rz_1.34_tight 2312ms → 907ms (2.5x)
- Speedup vs Run 4 (t=16): Rz_0.3_tight 2095ms → 718ms (2.9x), Rz_1.34_tight 1917ms → 601ms (3.2x)
- All distances preserved; 89/89 tests pass.

- Why 2.4x instead of 4-10x (estimated):
  Pairs reaching phase2_pq have already passed the outer Cauchy-Schwarz check, meaning
  |dot_outer| is close to threshold. Consequently, max_perp needs only a small contribution
  from z1,z0 to potentially bridge the gap, so the z2/z1 alignment prune fires only at the
  sphere boundary (small rem2), not near the center. Most z2/z1 iterations still enter the
  inner computation. Strategy B (algebraic replace) is needed for further speedup.
